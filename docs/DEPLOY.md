# Deploy

## Render (рекомендуется)

В репозитории есть `render.yaml` с двумя сервисами:

- `ttr-server` (`web`)
- `ttr-client` (`static_site`)

### Шаги

1. Открой Render и выбери Blueprint deploy.
2. Подключи репозиторий и ветку `main`.
3. Запусти деплой.

### Что важно

- `VITE_SERVER_URL` для клиента берется из URL сервера через `fromService`.
- Health endpoint сервера: `/health`.

## Docker (локально)

Используется `docker-compose.yml`:

```bash
docker compose up
```

Сервисы:

- server: `3000`
- client: `5173`

## Проверка после деплоя

1. Открыть клиентский URL.
2. Создать комнату.
3. Подключить второго игрока.
4. Проверить старт, ход, и обмен состоянием в realtime.

## Частые проблемы

- Неверный URL сервера в клиенте.
- Ошибки CORS/WS при нестандартной конфигурации proxy.
- Пропущенная сборка shared-пакета.
