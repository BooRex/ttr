# Gameplay

Документ описывает текущий игровой цикл и поддерживаемые механики.

## Базовый цикл

1. Хост создает комнату.
2. Игроки подключаются и хост запускает игру.
3. Игрок в свой ход выбирает одно действие.
4. После действия ход переходит следующему игроку.
5. Когда срабатывает условие финального раунда, игра завершается и считается итог.

## Доступные действия в ход

- взять карты поездов;
- взять destination-карты и выбрать, какие оставить;
- захватить маршрут;
- построить станцию (только Europe).

## Карта

- По умолчанию используется Europe.
- Поддерживаются типы маршрутов: normal, tunnel, ferry, double.
- Для tunnel/ferry применяется требование по минимуму локомотивов (упрощенная логика проекта).

## Параллельные маршруты (double)

- При 2-3 игроках второй параллельный путь недоступен, если один уже занят.
- При 4-5 игроках оба параллельных пути могут быть заняты.

## Станции (Europe)

- Игрок может строить станции в городах (ограниченный запас).
- В финальном подсчете станция может помочь закрыть destination через чужой путь.
- Неиспользованные станции дают бонусные очки (см. `SCORING_RULES.md`).

## Финал партии

Партия заканчивается после финального раунда, когда выполнено условие завершения в игровом движке.
Итоговая таблица учитывает:

- очки за захваченные маршруты,
- очки за выполненные/проваленные destination,
- бонусы (станции, самый длинный путь).
