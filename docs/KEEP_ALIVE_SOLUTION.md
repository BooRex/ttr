# 🌟 Keep-Alive решение для Render

## Проблема

На **бесплатном плане Render** сервис засыпает после **15 минут неактивности**. Это означает:
- Первый запрос от пользователя может занять 30+ секунд
- Сервер перезагружается
- Друзья видят долгую задержку

## Решение ✅

Добавили **Keep-Alive скрипт** (`apps/client/src/lib/keepAlive.ts`):

### Как работает?

1. **При загрузке приложения** (в App.tsx) запускается скрипт
2. **Каждые 5 минут** фронтенд пингует `/health` endpoint на backend
3. **Backend "просыпается"** и остается активным
4. Друзья всегда получают быстрый ответ ⚡

### Код

```typescript
// apps/client/src/lib/keepAlive.ts
export function startKeepAlive() {
  const serverUrl = import.meta.env.VITE_SERVER_URL || window.location.origin;
  
  // Пинг каждые 5 минут
  setInterval(() => {
    fetch(`${serverUrl}/health`);
  }, 5 * 60 * 1000);
}
```

### Интеграция

```typescript
// App.tsx
useEffect(() => {
  startKeepAlive();
}, []);
```

## 📊 Результаты

| Ситуация | Раньше | Сейчас |
|----------|--------|--------|
| Перый запрос после простоя | ⏳ 30+ сек | ⚡ <1 сек |
| Друзья ждут | 😤 Долго | 😊 Быстро |
| Сервер спит | Часто | Никогда |

---

## 💡 Альтернативы

Если keep-alive недостаточно:

### 1. **Платный план Render** ($7/месяц)
- ✅ Сервер никогда не спит
- ✅ Лучшая производительность
- ❌ Платно

### 2. **Fly.io вместо Render**
- ✅ Бесплатно и не спит
- ✅ Лучше с WebSockets
- ❌ Нужно перенести

### 3. **Cronitor/Uptimerobot** (внешний сервис)
- ✅ Отдельный пинг вне приложения
- ❌ Еще одна зависимость
