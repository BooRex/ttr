# Архитектура

## Компоненты

- `apps/client` — React UI, состояние клиента, рендер карты и панелей.
- `apps/server` — Socket.io/Express сервер, валидация и изменение игрового состояния.
- `packages/shared` — общие типы, определения карты, правила и утилиты подсчета.
- `calibrator` — отдельный инструмент ручной калибровки `europeCities`.

## Логические роли

### Клиент

- отправляет игровые действия через Socket.io;
- получает обновленное состояние комнаты;
- отображает игру, панели действий, события, результаты;
- управляет локальными UI-состояниями (выбор маршрута, подсветки и т.п.).

### Сервер

- хранит комнаты и состав участников;
- проверяет валидность действий в `gameEngine`;
- применяет правила хода и финального подсчета;
- рассылает состояние участникам комнаты.

### Shared

- типы `GameState`, `Route`, события клиента/сервера;
- карта Europe и связанные данные;
- функции правил/подсчета (например, очки за длину маршрута).

## Поток данных (high-level)

1. Игрок выполняет действие в UI.
2. Клиент отправляет событие (`room:*`/`game:*`).
3. Сервер валидирует и обновляет состояние.
4. Сервер отправляет обновление всем участникам комнаты.
5. Клиент перерисовывает UI по новому `game state`.

## Состояние и границы ответственности

- Истина по игровой логике находится на сервере.
- Клиент не принимает итоговые игровые решения, только инициирует действия.
- Для каждого viewer сервер маскирует скрытую информацию соперников (руки карт).

## Карта и геометрия

- Визуальный layout карты в `apps/client/src/lib/maps/europeLayout.ts`.
- Источник координат и label offset: `europeCities`.
- Правка координат выполняется через `calibrator` и переносится обратно в `europeLayout.ts`.
