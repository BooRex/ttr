# Ticket to Ride MVP - Roadmap на 3 спринта

Обновлено: 2026-03-28

## Текущий статус

Сейчас уже есть:
- гостевой вход по `sessionToken` + ник
- создание/вход в комнату, старт игры, наблюдатели
- реконнект
- базовые действия: `draw-card`, `claim-route`
- финальный раунд и завершение игры
- итоговый подсчет и таблица результатов

---

## Sprint 1 (4-8 часов) - Довести core-правила хода

### Цель
Сделать ход игрока ближе к классическим правилам Ticket to Ride и завершить обязательные серверные ограничения.

### Задачи
- [x] Реализовать действие `draw-destinations` на сервере
- [x] Правило выбора: взять 3, оставить минимум 1
- [x] Добавить событие Socket.io для подтверждения/возврата карт назначения
- [x] Добавить серверный опциональный таймер хода (из настроек комнаты)
- [x] При истечении таймера: автопропуск хода
- [x] Обновить UI для выбора destination-карт
- [x] Обновить тесты `GameEngine` (новое действие + таймерные edge-cases)

### Definition of Done
- [x] Игрок может брать карты назначения в середине партии
- [x] Таймер корректно работает и не ломает синхронизацию
- [x] После таймаута ход гарантированно переходит дальше
- [x] Все серверные тесты проходят

---

## Sprint 2 (6-10 часов) - Полная USA карта и валидный геймплей

### Цель
Перейти с mini-map к полной USA карте, чтобы партия ощущалась полноценной.

### Задачи
- [x] Заменить MVP mini-map на полный набор городов/маршрутов USA
- [x] Добавить полный набор destination-карт USA
- [x] Проверить и поправить баланс колоды/раздачи
- [x] Актуализировать проверки связности и подсчет очков под полную карту
- [x] Добавить smoke-тесты на типовые сценарии партии на полной карте

### Definition of Done
- [x] Можно сыграть полноценную партию USA от старта до финиша
- [x] Очки маршрутов и destination-карт считаются корректно
- [x] Нет утечки состояния между комнатами

---

## Sprint 3 (6-12 часов) - Mobile-first UI и удобство игры

### Цель
Сделать игру комфортной на телефоне (твой обязательный критерий).

### Задачи
- [x] Перейти от текстового MVP-board к интерактивному board-компоненту (Konva)
- [x] Реализовать touch-жесты: pan + zoom
- [x] Сделать мобильную раскладку: action bar + player info + hand panel
- [x] Добавить явную индикацию доступных действий текущего хода
- [x] Вынести ошибки/системные сообщения в понятные toast-уведомления

### Definition of Done
- [x] Играбельно с телефона в portrait и landscape
- [x] Клик/тап по маршрутам стабилен (hitStrokeWidth=28, ResizeObserver)
- [x] UI не требует desktop для базовых действий

---
