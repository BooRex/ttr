# План фиксов и улучшений (2026-04-30)

## Цель
Закрыть найденные баги и улучшения по итогам реальной игровой сессии, не ломая текущий цикл игры.

## Чеклист работ

- [ ] Фаза 0: критические баги игрового цикла
- [ ] Фаза 1: UX/визуал
- [ ] Фаза 2: правила/справка по очкам
- [ ] Фаза 3: стабилизация и проверка

---

## Фаза 0 — критические фиксы

1. Починить блокировку кнопки `x2` (добор карт из колоды).
2. Найти и исправить баг, когда станция строится не в выбранном городе.
3. Пересобрать финальный UX:
   - в панели `Игра завершена` оставить только кнопку `Лобби`;
   - кнопка `Лобби` делает `reload` страницы;
   - подробную статистику оставить только в `Итоговой таблице`;
   - в `Итоговой таблице` всем игрокам показывать очки всех игроков.

### Критерии приемки
- Кнопка `x2` активна/неактивна строго по правилам хода.
- Станция всегда строится в выбранный игроком город.
- На экране `Игра завершена` нет дублей статистики.
- В `Итоговой таблице` каждый игрок видит очки всех участников.

## Фаза 1 — UX/визуал

1. Добавить блок «Последние 2 взятые карты».
2. Назначить 5-му игроку лиловый цвет во всех местах UI.
3. Улучшить визуал выбора карт для постройки станции (четкий selected-state).
4. Для собранного маршрута добавить зеленую иконку «готово» справа.

### Критерии приемки
- Видно последние 2 взятые карты (по согласованным правилам видимости).
- Цвет 5-го игрока стабильно лиловый во всех компонентах.
- Выбранные карты станции явно читаются.
- Собранные маршруты имеют зеленый индикатор.

---

## Фаза 2 — правила и справка по очкам

1. Обновить таблицу/модалку очков: добавить пункт про маршруты длиной `>6`.
2. Синхронизировать текст в локалях (`ru`, `en`, `uk`, `de`).

### Критерии приемки
- В справке есть явное правило для длин `>6`.
- Во всех языках строка присутствует и совпадает по смыслу.

---

## Фаза 3 — стабилизация

1. Добавить/обновить тесты на:
   - логику `x2`;
   - валидацию станции по городу;
   - индикатор выполнения маршрута;
   - финальный экран без дублей статистики.
2. Провести ручной smoke-test с 2–5 игроками.

### Критерии приемки
- Базовый сценарий игры проходит без блокеров.
- Ключевые баги не воспроизводятся повторно.

---

## Риски и уточнения

1. Видимость блока «Последние 2 карты»:
   - только для текущего игрока;
   - или для всех (с ограничением деталей).
2. Точное правило начисления очков за маршруты длиной `>6`.
3. Причина бага станции может быть как в UI-состоянии, так и в серверной синхронизации.
