## Технический план по пунктам

### 1) Комбинации захвата (цвет + локомотивы)
- Проверить и выровнять серверную валидацию в `apps/server/src/gameEngine.ts`.
- В `apps/client/src/components/ActionPanel.tsx` показывать полный список валидных комбинаций оплаты для выбранного маршрута.
- Для серых маршрутов показывать несколько вариантов (цвет + N локомотивов, pure locomotive).
- Добавить тесты в `apps/server/test/gameEngine.test.ts`.

### 2) Карта не должна уезжать бесконечно
- В `apps/client/src/components/BoardCanvas.tsx` добавить clamp позиции в зависимости от текущего `scale` и размеров контейнера.
- Применять clamp после drag, wheel zoom и pinch zoom.
- Оставить небольшой "люфт" (например 12-20px), чтобы UX не был слишком жестким.

### 3) Карта Европы + поддержка кастомных карт
- Создать `packages/shared/src/maps/europe.ts`.
- Перенести геометрию карты (города, координаты, routes layout) в map-definition вместо жесткого USA внутри `BoardCanvas`.
- Обновить `packages/shared/src/index.ts` и выбор карты в `apps/client/src/App.tsx`.
- Цель: единый формат, чтобы позже легко добавить кастомные карты.

### 4) Локализация (uk/en/de/ru)
- Подключить i18n слой на клиенте (например, `i18next` + `react-i18next`).
- Вынести UI-строки из `App.tsx`, `ActionPanel.tsx` и смежных компонентов в словари.
- Для игровых событий использовать ключи + параметры (не готовые строки), чтобы переводить на клиенте.

### 5) Панель игроков + цвета
- Зафиксировать цвета игроков в порядке: red/blue/green/yellow/black.
- Заменить зависимость от tooltip на всегда видимую компактную инфо-панель (очки, вагоны, карты, индикатор хода).
- Улучшить доступность на мобильных: крупные tap-target, контраст текста на yellow/green.

### 6) Звук + анимация "мой ход"
- Добавить звуковой сигнал при переходе хода к текущему пользователю.
- Добавить короткий визуальный эффект (пульс/свечение HUD + карточка "Ваш ход").
- Учесть autoplay-policy: активировать звук после первого взаимодействия пользователя.

### 7) Анимация захвата маршрута
- При успешном захвате запускать анимацию на маршруте: `dash -> solid`, затем короткий highlight.
- Сделать эффект заметным, но легким по производительности (без тяжелых blur на всем холсте).

### 8) Подсветка маршрутов игрока из topbar
- Hover (desktop) / tap-toggle (mobile) на иконке игрока.
- На карте подсвечивать все маршруты этого игрока, остальные приглушать.
- Повторный tap снимает фильтр.

### 9) Базовая графика карты (море/земля)
- Добавить фоновый слой в `BoardCanvas`: векторные формы или легкий raster.
- Сначала минимальный стиль (land/sea), затем декоративные элементы.
- Проверить FPS на мобильных устройствах.

### 10) Куда перенести события
- Серверный исторический лог оставить в `state.log`.
- Клиентские эффекты (toast/sound/animations) вынести в отдельный event-layer (store/hook/service).
- Socket subscriptions вынести из `App.tsx` в отдельный модуль/хук для упрощения поддержки.

## Риски и зависимости
- Производительность `react-konva` при комбинации нескольких анимаций/подсветок.
- Автозвук может блокироваться браузером без пользовательского действия.
- Без рефакторинга событий возможны дубли (лог + toast + звук).
- Мультикарта требует отвязки `BoardCanvas` от USA-координат до масштабирования контента.
