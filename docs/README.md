# Документация проекта

Этот каталог содержит только актуальные документы, нужные для разработки и поддержки проекта.

## Карта документации

- `GETTING_STARTED.md` — установка, запуск, структура workspace
- `ARCHITECTURE.md` — архитектура frontend/server/shared и поток данных
- `API.md` — Socket.io события и их назначение
- `GAMEPLAY.md` — игровой цикл и поддерживаемые механики
- `SCORING_RULES.md` — подсчет очков и финальный результат
- `DEPLOY.md` — деплой на Render и запуск через Docker
- `ROADMAP.md` — ближайший план развития

## Критерии, по которым документы оставлены

- документ описывает текущую систему, а не временный план;
- документ нужен новому разработчику для входа в проект;
- документ содержит правила, интерфейсы или эксплуатационные шаги;
- документ не дублирует другой файл с той же информацией.

## Что обязательно поддерживать в актуальном состоянии

1. Изменения Socket-событий и payload → обновлять `API.md`.
2. Изменения игровых правил/механик → обновлять `GAMEPLAY.md` и `SCORING_RULES.md`.
3. Изменения в запуске/скриптах/infra → обновлять `GETTING_STARTED.md` и `DEPLOY.md`.
4. Сильные архитектурные изменения (новые слои, крупный рефактор) → обновлять `ARCHITECTURE.md`.
5. Новый этап продукта/приоритеты → обновлять `ROADMAP.md`.
