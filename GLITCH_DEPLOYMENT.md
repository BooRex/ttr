# Деплой на Glitch

## Шаг за шагом

### 1️⃣ Подготовка GitHub репозитория

```bash
# Убедись, что все закомичено
git add .
git commit -m "Prepare for Glitch deployment"
git push origin main
```

### 2️⃣ На сайте Glitch

1. Перейди на https://glitch.com
2. Нажми "New Project" → "Import from GitHub"
3. Авторизуй GitHub (если нужно)
4. Выбери репо `ticket-to-ride-mvp`
5. Подожди, пока Glitch скачает проект (~2-3 минуты)

### 3️⃣ Настройка Glitch

После импорта:

```bash
# Откройте Terminal (Tools → Terminal)
cd /app

# 1. Установи зависимости
npm install

# 2. Собери проект
npm run build

# 3. Запусти
npm run start -w apps/server
```

**Автозапуск не пройдет** - нужно вручную настроить.

### 4️⃣ Настройка автозапуска

1. Открой `.glitchignore` (или создай):

```
node_modules/
dist/
.env.local
.DS_Store
```

2. Открой `package.json` в корне и измени start скрипт:

```json
{
  "scripts": {
    "start": "npm run build && npm run start -w apps/server",
    "dev": "npm run dev:all"
  }
}
```

### 5️⃣ Переменные окружения

Глич их устанавливает автоматически - не нужно ничего делать.

### 6️⃣ Получи URL

После того как сервер запустился:
- Глич выдаст URL типо: `https://project-name.glitch.me`
- Можешь поделиться этой ссылкой с друзьями

---

## ⚠️ Важно для фронтенда

Фронтенд нужно собрать ОДИН РАЗ и сервировать статично.

В файле `apps/server/src/main.ts` убедись, что есть:

```typescript
// Сервировать фронтенд
app.use(express.static('../client/dist'));
app.get('*', (req, res) => {
  res.sendFile('../client/dist/index.html');
});
```

## 🐛 Если что-то не работает

Открой **Logs** в Гличе и посмотри ошибки.

Частые проблемы:

| Проблема | Решение |
|----------|---------|
| `Cannot find module` | Запусти `npm install` в терминале |
| `Port already in use` | Перезагрузи приложение (restart) |
| Фронтенд не загружается | Проверь, что `dist` собран |
| WebSocket не работает | Проверь CORS в main.ts |
